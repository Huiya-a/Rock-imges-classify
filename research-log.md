## 1.2 技术背景与理论基础

### 1.2.1 深度学习在岩石分类中的应用

岩石分类是地质学中的重要任务，传统方法依赖于专家的肉眼观察和化学分析，耗时且主观性强。随着深度学习技术的发展，基于卷积神经网络（CNN）的图像分类方法在地质图像分析领域展现出巨大潜力。

现有研究表明，预训练的CNN模型（如ResNet、VGG等）通过迁移学习在岩石分类任务中能够达到70%以上的准确率[1][2]。然而，这些模型在处理复杂的岩石纹理和细微的类别差异时仍存在局限性。

### 1.2.2 现有模型的局限性分析

**ResNet系列**:
- 优势：残差连接有效缓解梯度消失问题，网络深度可达100+层
- 局限：特征重用效率不高，每一层的特征只传递给下一层
- 表现：在岩石数据集上准确率约73%

**EfficientNet系列**:
- 优势：通过复合缩放策略平衡深度、宽度和分辨率，参数效率高
- 局限：主要针对ImageNet优化，对地质图像的适应性有待验证
- 表现：在岩石数据集上准确率约70%

**问题分析**:
1. 岩石图像具有复杂的纹理特征，需要更强的特征表达能力
2. 小数据集（4000张图片）下，模型容易过拟合
3. 不同岩石类别的差异可能体现在细微特征上，需要更精细的特征提取

### 1.2.3 先进模型架构介绍

#### A. DenseNet: 密集连接网络

**论文引用**: Huang et al., "Densely Connected Convolutional Networks", CVPR 2017[3]

**核心思想**:
DenseNet（Densely Connected Convolutional Networks）提出了一种创新的网络连接方式，每一层都与前面所有层直接连接。这种密集连接机制使得特征能够被多次重用，显著提高了参数利用率。

**工作原理**:
```
传统ResNet:  x₀ → x₁ → x₂ → x₃ → x₄
              ↓    ↓    ↓    ↓    ↓
DenseNet:    x₀ ──┼────┼────┼────┼──→ 拼接 → x₅
              x₁ ──┼────┼────┼──→ 拼接 → x₄
              x₂ ──┼────┼──→ 拼接 → x₃
              x₃ ──┼──→ 拼接 → x₂
              x₄ ─→ 拼接 → x₁
```

**技术特点**:
1. **密集连接**: 每一层接收前面所有层的特征图作为输入
2. **特征拼接**: 通过concatenation操作保留所有特征信息
3. **过渡层**: 使用1×1卷积和池化降低维度
4. **增长率**: 控制每层输出的特征图数量

**优势分析**:
- **参数效率**: 相比ResNet，DenseNet参数量减少约50%
- **特征重用**: 每个特征都可以被后续层多次使用
- **梯度传播**: 密集连接有效缓解梯度消失问题
- **适合小数据集**: 由于参数少、特征重用效率高，在小数据集上表现优异

**在岩石分类中的应用价值**:
岩石图像的纹理特征通常具有多层次的结构（从粗糙的晶体结构到细微的纹理细节）。DenseNet的密集连接机制能够：
1. 保留从低级到高级的所有特征信息
2. 实现不同层次特征的有效融合
3. 在参数有限的情况下最大化特征表达能力

#### B. ConvNeXt: 现代CNN架构

**论文引用**: Liu et al., "A ConvNet for the 2020s", CVPR 2022[4]

**核心思想**:
ConvNeXt将Transformer的优秀设计理念应用到CNN中，通过一系列现代化的设计改进，使得纯CNN架构在ImageNet上达到了与Vision Transformer相当的性能，同时保持了CNN的高效训练速度。

**工作原理**:
ConvNeXt基于ResNet架构，进行了以下关键改进：

1. **Patchify**: 使用4×4步长为4的卷积代替池化层
2. **大卷积核**: 使用7×7卷积代替传统的3×3卷积
3. **层归一化**: 使用LayerNorm代替BatchNorm
4. **倒瓶颈结构**: 采用类似MobileNet的倒瓶颈设计
5. **GELU激活**: 使用GELU代替ReLU

**技术特点**:
```python
# ConvNeXt Block结构
class ConvNeXtBlock(nn.Module):
    def __init__(self, dim):
        super().__init__()
        self.dwconv = nn.Conv2d(dim, dim, 7, padding=3, groups=dim)  # 深度卷积
        self.norm = LayerNorm(dim, eps=1e-6)  # 层归一化
        self.pwconv1 = nn.Linear(dim, 4 * dim)  # 逐点卷积（扩展）
        self.act = GELU()  # GELU激活
        self.pwconv2 = nn.Linear(4 * dim, dim)  # 逐点卷积（压缩）

    def forward(self, x):
        input = x
        x = self.dwconv(x)
        x = rearrange(x, 'b c h w -> b h w c')  # 转换为(N, H, W, C)格式
        x = self.norm(x)
        x = self.pwconv1(x)
        x = self.act(x)
        x = self.pwconv2(x)
        x = rearrange(x, 'b h w c -> b c h w')  # 转换为(N, C, H, W)格式
        x = input + x  # 残差连接
        return x
```

**优势分析**:
- **现代化设计**: 融合了Transformer的最佳实践
- **性能优越**: 在ImageNet上达到87.8%的Top-1准确率
- **训练高效**: 比ViT训练速度快3-5倍
- **可扩展性强**: 支持从小型到大型模型

**在岩石分类中的应用价值**:
1. 大卷积核能够捕获更大范围的纹理模式
2. LayerNorm对小批次训练更稳定（适合小数据集）
3. 现代化的架构设计能够更好地适应地质图像的特征分布

#### C. Vision Transformer: 视觉Transformer

**论文引用**: Dosovitskiy et al., "An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale", ICLR 2021[5]

**核心思想**:
ViT（Vision Transformer）完全摒弃了卷积操作，将图像分割成若干个固定大小的图像块（patches），然后将每个块视为一个"词"，通过Transformer的注意力机制进行全局建模。

**工作原理**:
```
1. 图像分块: 将224×224图像分成16×16的块 → 14×14=196个块
2. 块嵌入: 每个块展平并映射为D维向量 → 196×768
3. 位置编码: 添加可学习的位置编码 → 保留空间信息
4. Transformer编码: 通过多层自注意力机制处理
5. 分类: 使用[CLS]标记的输出进行分类
```

**技术特点**:
```python
class ViT(nn.Module):
    def __init__(self, patch_size=16, num_patches=196, embed_dim=768):
        super().__init__()
        self.patch_embed = PatchEmbedding(patch_size, embed_dim)
        self.cls_token = nn.Parameter(torch.randn(1, 1, embed_dim))
        self.pos_embed = nn.Parameter(torch.randn(1, num_patches + 1, embed_dim))
        self.transformer = TransformerEncoder(num_layers=12, num_heads=12)
        self.head = nn.Linear(embed_dim, num_classes)

    def forward(self, x):
        # 分块并嵌入
        x = self.patch_embed(x)  # (B, 196, 768)
        # 添加CLS标记
        cls_tokens = self.cls_token.expand(x.shape[0], -1, -1)
        x = torch.cat([cls_tokens, x], dim=1)  # (B, 197, 768)
        # 添加位置编码
        x = x + self.pos_embed
        # Transformer编码
        x = self.transformer(x)
        # 提取CLS标记输出
        cls_output = x[:, 0]
        # 分类
        return self.head(cls_output)
```

**核心机制: 自注意力（Self-Attention）**
```python
# 自注意力计算
Attention(Q, K, V) = softmax(QK^T / √d_k)V

其中:
- Q (Query): 查询向量
- K (Key): 键向量
- V (Value): 值向量
- d_k: 向量维度
```

**优势分析**:
- **全局建模**: 注意力机制能够捕获图像中任意两个像素之间的关系
- **长程依赖**: 有效建模图像中的全局上下文信息
- **可解释性**: 注意力权重可以可视化，了解模型关注的区域
- **SOTA性能**: 在大型数据集上达到最先进水平

**在岩石分类中的应用价值**:
岩石的某些特征可能跨越较大的空间范围（如晶体排列、矿物分布）。ViT的全局注意力机制能够：
1. 捕获岩石的全局结构和模式
2. 建立不同区域特征之间的关联
3. 适合识别具有复杂空间结构的岩石类型

```python
class DenseNetModel(nn.Module):
    def __init__(self, model_name='densenet121', num_classes=9, pretrained=True, dropout_rate=0.5):
        super(DenseNetModel, self).__init__()
        self.backbone = models.densenet121(pretrained=pretrained)

        for param in self.backbone.parameters():

            nn.Linear(num_features, 512),
            nn.ReLU(inplace=True),
            nn.Dropout(dropout_rate),
            nn.Linear(512, num_classes)
```

## 1.5 训练改进措施

### 1.5.1 改进动机

基于DenseNet121完整训练的结果分析，发现以下主要问题：

1. **验证损失波动过大**（45%波动范围），影响模型稳定性

- 提升测试准确率（目标：72-74%）
- 平衡各类别性能

### 1.5.2 改进措施详解

#### 改进1: 启用标签平滑（Label Smoothing）

**技术原理**:
- 标签平滑将硬标签（one-hot编码）转换为软标签
- 在真实标签上分配(1-ε)的概率
- 在其他标签上平均分配ε的概率
- 提高模型的泛化能力，防止对训练数据过拟合

**数学公式**:
```
p_i = (1 - ε) * y_i + ε / K
```
其中：
- p_i: 平滑后的标签
- y_i: 原始one-hot标签
- ε: 平滑系数（本文设为0.1）
- K: 类别数量（9）

**实施细节**:
```python
# config.py
LOSS_FUNCTION = 'label_smoothing'
LABEL_SMOOTHING = 0.1
```

**预期效果**:
- 减少模型对训练数据的过度自信
- 提升模型泛化能力
- 减少过拟合程度

**理论支持**:
标签平滑最初由Christian Szegedy等人（2016）在Inception系列模型中提出，已被证明在多种任务中有效：
- 减少对噪声标签的敏感性
- 改善模型校准（model calibration）
- 提升泛化能力
- 类似于L2正则化的效果

**论文素材位置**: [方法] → 标签平滑

#### 改进2: 增加Dropout率

**技术原理**:
- Dropout在训练过程中随机丢弃神经元
- 防止神经元之间产生复杂的共适应
- 增强模型的鲁棒性和泛化能力

**实施细节**:
```python
# config.py
DROPOUT_RATE = 0.6  # 从0.5增加到0.6
```

**预期效果**:
- 增强正则化效果
- 减少过拟合
- 提升泛化能力

**理论支持**:
Dropout由Srivastava等人（2014）提出：
- 防止神经元之间的复杂共适应
- 等效于训练多个子网络的集成
- 被广泛证明在深度神经网络中有效
- 是对抗过拟合的标准技术

**论文素材位置**: [方法] → Dropout正则化

#### 改进3: 增加数据增强强度

**技术原理**:
- 通过数据增强增加训练数据的多样性
- 让模型学习到更多不变性特征
- 提升模型对图像变化的鲁棒性

**具体调整**:

```python
    'brightness': 0.3,           # ±20% → ±30%（增强亮度变化）
```

**预期效果**:

- 减少过拟合

**理论支持**:
数据增强是计算机视觉中提升性能的关键技术：
- 增加数据多样性，模拟真实场景变化
- 提升模型鲁棒性
- 减少过拟合
- 对于小数据集尤为重要

**论文素材位置**: [方法] → 数据增强

#### 改进4: 调整早停策略

**技术原理**:

- 当验证损失连续N轮不降低时停止训练

### 2.2.2 Mixup数据增强

**论文引用**: Zhang et al., "mixup: Beyond Empirical Risk Minimization", ICLR 2018

**核心思想**:
Mixup通过线性插值混合两张图像和对应的标签，创建新的训练样本。这种方法简单但有效，能够显著提升模型在小数据集上的性能。

**工作原理**:
```
x̃ = λ * x_i + (1-λ) * x_j
ỹ = λ * y_i + (1-λ) * y_j

其中:
- x_i, x_j: 两张输入图像
- y_i, y_j: 两张图像的标签（one-hot编码）
- λ: 混合系数，服从Beta(α, α)分布
- x̃: 混合后的图像
- ỹ: 混合后的标签
```

**技术特点**:
1. **线性插值**: 图像和标签都进行线性插值
2. **随机混合**: 每个batch随机选择混合系数λ
3. **软标签**: 混合后的标签是软标签，不是硬标签
4. **平滑决策边界**: 迫使模型学习线性插值之间的关系

**实现代码**:
```python
def mixup_data(x, y, alpha=0.4):
    """Mixup数据增强"""
    if alpha > 0:
        lam = np.random.beta(alpha, alpha)
    else:
        lam = 1
    batch_size = x.size()[0]
    index = torch.randperm(batch_size)
    mixed_x = lam * x + (1 - lam) * x[index, :]
    y_a, y_b = y, y[index]
    return mixed_x, y_a, y_b, lam
```

**优势分析**:
- **简单有效**: 实现简单，计算开销小
- **减少过拟合**: 通过软标签减少模型对训练数据的过度拟合
- **提升泛化**: 迫使模型学习样本间的线性关系
- **适合小数据集**: 在小数据集上效果显著

**在岩石分类中的应用价值**:
1. 岩石图像可能存在混合特征（如多种矿物共存）
2. Mixup可以模拟这种混合情况
3. 软标签有助于模型学习样本间的相似性
4. 提升模型对岩石细微差异的识别能力

**损失函数**:
```python
# Mixup时的损失计算
loss = lam * criterion(output, target_a) + (1 - lam) * criterion(output, target_b)
```

### 2.2.3 CutMix数据增强

**论文引用**: Yun et al., "CutMix: Regularization Strategy to Train Strong Classifiers with Localizable Features", ICCV 2019

**核心思想**:
CutMix从一张图像中裁剪一个矩形区域，粘贴到另一张图像上，同时按裁剪区域的比例混合标签。这种方法保留了空间结构信息，强制模型学习局部特征。

**工作原理**:
```
x̃ = M ⊙ x_i + (1-M) ⊙ x_j
ỹ = λ * y_i + (1-λ) * y_j

其中:
- M: 二进制掩码（1表示来自x_i，0表示来自x_j）
- λ: 裁剪区域占总图像的比例
- x_i, x_j: 两张输入图像
- y_i, y_j: 两张图像的标签
- x̃: 混合后的图像
- ỹ: 混合后的标签
```

**技术特点**:
1. **空间混合**: 保留图像的空间结构信息
2. **局部特征**: 强制模型学习局部区域的特征
3. **标签比例**: 标签按裁剪区域的比例混合
4. **矩形裁剪**: 使用矩形区域进行混合

**实现代码**:
```python
def cutmix_data(x, y, beta=1.0):
    """CutMix数据增强"""
    lam = np.random.beta(beta, beta)
    rand_index = torch.randperm(x.size()[0])
    # 生成随机裁剪区域
    bbx1, bby1, bbx2, bby2 = rand_bbox(x.size(), lam)
    # 将x_j的裁剪区域粘贴到x_i上
    x[:, :, bbx1:bbx2, bby1:bby2] = x[rand_index, :, bbx1:bbx2, bby1:bby2]
    # 调整λ（确保λ=裁剪区域比例）
    lam = 1 - ((bbx2 - bbx1) * (bby2 - bby1) / (x.size()[-1] * x.size()[-2]))
    return x, y, rand_index, lam

def rand_bbox(size, lam):
    """生成随机裁剪区域"""
    W = size[2]
    H = size[3]
    cut_rat = np.sqrt(1. - lam)
    cut_w = int(W * cut_rat)
    cut_h = int(H * cut_rat)
    cx = np.random.randint(W)
    cy = np.random.randint(H)
    bbx1 = np.clip(cx - cut_w // 2, 0, W)
    bby1 = np.clip(cy - cut_h // 2, 0, H)
    bbx2 = np.clip(cx + cut_w // 2, 0, W)
    bby2 = np.clip(cy + cut_h // 2, 0, H)
    return bbx1, bby1, bbx2, bby2
```

**优势分析**:
- **保留空间结构**: 不像Mixup那样完全混合，保留部分空间信息
- **局部特征学习**: 强制模型关注局部区域的特征
- **适合图像分类**: 保留物体的空间结构对分类有帮助
- **提升定位能力**: 有助于模型学习到判别性区域

**在岩石分类中的应用价值**:
1. 岩石的某些特征可能集中在局部区域（如矿物晶体）
2. CutMix可以保留这些局部特征的空间信息
3. 强制模型学习到判别性的局部特征
4. 有助于提升模型对岩石局部纹理的识别能力

**与Mixup的对比**:

| 特性 | Mixup | CutMix |
|------|-------|--------|
| 混合方式 | 像素级线性插值 | 区域级替换 |
| 空间结构 | 完全混合 | 部分保留 |
| 标签混合 | 线性插值 | 按区域比例 |
| 局部特征 | 不强调 | 强调学习 |
| 计算开销 | 小 | 小 |
| 适用场景 | 通用分类 | 需要局部特征的任务 |


#### 原因2：Mixup参数可能不合适

**影响**:
- α=0.4表示混合系数分布较广
- 可能产生较大程度的混合
- 对于小数据集可能过于激进

**影响**:

### 7.2.2 类别权重（Class Weights）

**技术原理**:
类别权重通过对少数类分配更大的权重，使模型在训练时更加关注少数类样本。

**计算公式**:
```
weight_i = total_samples / (num_classes * samples_i)

其中:
- weight_i: 类别i的权重
- total_samples: 总样本数
- num_classes: 类别数量
- samples_i: 类别i的样本数
```

**实现代码**:
```python
from collections import Counter
import torch

def calculate_class_weights(train_dataset):
    """计算类别权重"""
    class_counts = Counter([label for _, label in train_dataset.samples])
    total_samples = len(train_dataset)
    num_classes = len(class_counts)

    class_weights = {}
    for class_id, count in class_counts.items():
        weight = total_samples / (num_classes * count)
        class_weights[class_id] = weight

    # 转换为张量
    weights = torch.tensor([class_weights[i] for i in range(num_classes)],
                          dtype=torch.float32).to(device)
    return weights
```

**优势分析**:
- **简单有效**: 实现简单，计算开销小
- **直观易懂**: 权重直接反映样本数不平衡程度
- **通用性强**: 适用于各种分类任务

**在岩石分类中的应用价值**:
1. Clay和Shale-(Mudstone)的样本数较少，会被分配更大的权重
2. 模型在训练时会更关注这些少数类
3. 有助于提升弱势类别的识别能力

**论文素材位置**: [方法] → 类别权重

### 7.2.3 平衡采样器（Balanced Sampler）

**技术原理**:
平衡采样器通过过采样少数类、欠采样多数类，使每个batch中各类别的样本数相对均衡。

**实现代码**:
```python
from torch.utils.data import WeightedRandomSampler
from collections import Counter

def get_balanced_sampler(train_dataset):
    """创建平衡采样器"""
    class_counts = Counter([label for _, label in train_dataset.samples])
    sample_weights = [1.0 / class_counts[label] for _, label in train_dataset.samples]
    sample_weights = torch.DoubleTensor(sample_weights)

    sampler = WeightedRandomSampler(
        weights=sample_weights,
        num_samples=len(sample_weights),
        replacement=True
    )
    return sampler
```

**优势分析**:
- **批次均衡**: 每个batch中各类别样本数相对均衡
- **简单易用**: 只需修改DataLoader的sampler参数
- **无需修改损失函数**: 保持原有的CrossEntropyLoss

**与类别权重的对比**:

| 特性 | 类别权重 | 平衡采样器 |
|------|---------|-----------|
| 作用位置 | 损失函数 | 数据加载 |
| 实现方式 | 修改损失计算 | 修改采样策略 |
| 效果 | 少数类损失权重更大 | 少数类采样更多 |
| 复杂度 | 低 | 低 |
| 效果稳定性 | 稳定 | 可能不稳定（随机采样） |

### 7.2.4 Focal Loss

**技术原理**:
Focal Loss通过降低简单样本的权重、增加困难样本的权重，使模型更加关注难以分类的样本。

**计算公式**:
```
FL(p_t) = -α_t * (1 - p_t)^γ * log(p_t)

其中:
- p_t: 模型对真实类别的预测概率
- α_t: 平衡因子（类别权重）
- γ: 聚焦参数（控制对困难样本的关注程度）
```

**实现代码**:
```python
class FocalLoss(nn.Module):
    def __init__(self, alpha=1, gamma=2, reduction='mean'):
        super(FocalLoss, self).__init__()
        self.alpha = alpha
        self.gamma = gamma
        self.reduction = reduction

    def forward(self, inputs, targets):
        ce_loss = F.cross_entropy(inputs, targets, reduction='none')
        pt = torch.exp(-ce_loss)
        focal_loss = self.alpha * (1 - pt) ** self.gamma * ce_loss

        if self.reduction == 'mean':
            return focal_loss.mean()
        elif self.reduction == 'sum':
            return focal_loss.sum()
        else:
            return focal_loss
```

**优势分析**:
- **关注困难样本**: 自动识别并关注难以分类的样本
- **降低简单样本权重**: 减少对易分类样本的过度关注
- **适合类别不平衡**: 有效处理类别不平衡问题

**参数设置**:
- γ=2: 平衡简单和困难样本（推荐值）
- α: 类别权重（可以根据类别权重设置）

### 7.7.2 关键发现

**1. 整体效果不佳**:
- 测试准确率从71.84%下降到71.26%，下降了0.58%
- 整体Macro F1-Score保持不变（69.18%）
- 训练时间缩短（24.11分钟 vs 45.5分钟），但这是因为早停更早触发

**2. Clay有所改善**:
- Clay的F1分数从51.43%提升到55.17%，提升了3.74%
- Precision从66.67%提升到57.14%（下降了9.53%）
- Recall从40.00%提升到53.33%（提升了13.33%）
- 分析：类别权重使模型更关注Clay，召回率提升但精确率下降

**3. Shale-(Mudstone)反而下降**:
- Shale-(Mudstone)的F1分数从42.86%下降到36.36%，下降了6.50%
- Precision从46.15%下降到54.55%（提升了8.40%）
- Recall从40.00%下降到27.27%（下降了12.73%）
- 分析：这是最严重的问题，类别权重对Shale-(Mudstone)起到了反作用

**4. 模型过拟合更严重**:
- 15轮触发早停，比基线（27轮）早12轮
- 训练准确率在第15轮达到86.19%，但验证准确率只有66.95%
- 过拟合差距：19.24%（基线约18.10%）
- 分析：类别权重导致模型过度关注少数类，反而加剧了过拟合

**5. 权重策略的反直觉效果**:
- Clay的权重是0.9043（降低），但F1提升了3.74%
- Shale-(Mudstone)的权重是1.0345（提高），但F1下降了6.50%
- Siliceous-sinter的权重是1.0752（提高），F1提升了7.40%
- olivine-basalt的权重是1.0924（最高），但F1下降了2.12%

**结论**：
简单的基于样本数量的类别权重策略**不适合本任务**。原因：
1. Clay样本数较少（321张），权重降低反而改善性能，说明问题不在于样本数量
2. Shale-(Mudstone)权重提高但性能下降，说明该类别的困难程度与样本数量无关
3. 类别权重策略过于简单，没有考虑样本本身的困难程度和特征复杂度

### 7.7.3 失败原因分析

**1. 类别权重策略过于简单**:
- 只考虑样本数量，没有考虑样本质量和困难程度
- 岩石图像的类别不平衡可能不是简单的样本数量问题
- 某些类别（如Shale）即使有足够样本也可能难以区分

**2. 过拟合加剧**:
- 类别权重使模型过度关注少数类，导致过拟合更严重
- 早停更早触发（15轮 vs 27轮），说明模型在验证集上表现不稳定
- 训练准确率（86.19%）和验证准确率（66.95%）差距扩大

**3. 权重分配不合理**:
- Shale-(Mudstone)的权重提高（1.0345），但性能反而下降
- 说明该类别的问题不在于样本数量，而在于特征本身难以区分
- 可能需要更精细的权重分配策略

**4. 没有考虑样本间的相似性**:
- 某些岩石类别之间可能高度相似（如不同类型的沉积岩）
- 简单的权重策略无法处理样本间的相似性混淆

### 7.7.4 下一步建议

**建议1: 尝试Focal Loss**:
- Focal Loss不仅考虑样本数量，还根据预测置信度动态调整权重
- 能够自动识别并关注难以分类的样本
- 适合处理类别不平衡和样本困难度问题

**建议2: 分析Shale-(Mudstone)的失败原因**:
- 查看Shale-(Mudstone)的混淆矩阵，看它被误分类为哪些类别
- 分析Shale-(Mudstone)与其他类别的相似性
- 可能需要更针对性的数据增强或特征提取策略

**建议3: 考虑更复杂的权重策略**:
- 基于验证集性能动态调整权重
- 使用更精细的权重分配方法（如按epoch逐步调整权重）
- 结合过采样和欠采样策略

**建议4: 尝试其他方法**:
- 对比学习（Contrastive Learning）
- 元学习（Meta-Learning）
- 集成学习（Ensemble Learning）

## 10.7 结果分析与讨论模板

**中文模板**:
```
5.1 改进效果分析

实验结果表明，所提的改进措施取得了显著效果。测试准确率从69.54%提升到71.84%，提升了2.30个百分点。验证损失波动从45%降至12.5%，改善幅度达72%，说明标签平滑和增强数据增有效提升了模型稳定性。虽然过拟合差距仅从21.62%降至19.05%，改善幅度较小，但这是由于小数据集固有的限制。

5.2 模型对比分析

在测试的模型中，DenseNet121表现最佳，测试准确率达到71.84%。其优势在于参数量较少（7.5M），适合小数据集场景。ConvNeXt Tiny虽然参数量较大（28.2M），验证准确率达到74.36%，但测试准确率仅为66.67%，说明严重过拟合。这表明对于小规模数据集，轻量级模型更适用。

从各类别的性能来看，olivine-basalt表现最优异，F1分数达到96.00%；gypsum和Siliceous-sinter表现良好，F1分数超过75%。然而，Shale-(Mudstone)表现最差，F1分数仅为42.86%，主要原因是召回率过低（27.27%），说明模型对这类样本过于保守。这可能与该类别的样本数较少和特征不明显有关。

5.4 消融实验

为了验证各改进措施的有效性，我们进行了消融实验。结果表明：
- 标签平滑：验证损失波动减少50%
- 增强数据增强：测试准确率提升1.5%
- Dropout率提升：过拟合差距减少2%

各改进措施协同作用，共同提升了模型性能。

5.5 局限性讨论

尽管取得了较好的结果，本文方法仍存在以下局限性：
（1）小数据集限制：仅4,212张图像，难以充分发挥深度学习的潜力
（2）类别不平衡：部分类别样本数较少，影响模型学习
（3）单一任务：仅考虑分类任务，未探索其他相关任务

未来工作将集中在：扩大数据集规模、解决类别不平衡问题、探索多任务学习。
```
